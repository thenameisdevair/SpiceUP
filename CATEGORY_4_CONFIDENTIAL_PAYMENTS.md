# Category 4 — Confidential Payments (Tongo) (Detailed Plan)

> **Goal**: Users can send and receive private transfers where amounts are hidden on-chain via ZK proofs.

---

## Context

- Categories 1–3 are complete: Privy auth works, Starkzap wallet is wired, public balances poll every 15s, Tongo private key is generated and stored in SecureStore on first login.
- `TongoConfidential` instance is already created in `hooks/useAuthInit.ts` and stored in `useAuthStore.tongo`.
- `ConfidentialState` (balance + pending) is fetched once on mount in `hooks/useConfidentialBalance.ts` and stored in `useWalletStore.confidential`.
- `constants/network.ts` has `tongoContract: "0x0"` as a placeholder — fill in when the real Tongo contract address is known. All code guards against `"0x0"` and degrades gracefully.
- This category adds the actual **transactional** capabilities on top of that foundation.

---

**File**: `lib/tongo.ts` — EXTEND (existing file has `generateTongoPrivateKey`, `getOrCreateTongoKey`, `initTongo`)

**Token support**: All helpers accept any `Token` (STRK, ETH, USDC). Token is embedded in `Amount.parse(amountStr, token)` so no extra `token` field is needed in the SDK detail objects.

## 4.2 Send Flow (Private)

> PRD: Recipient's Tongo address (base58 or QR scan), amount + token selector, "amount will be hidden on-chain" indicator, `confidentialTransfer`, preflight before submitting.

**File**: `app/(app)/send.tsx` — EXTEND (existing public-only send screen)

### Changes:

**Mode toggle** — segmented control at top of screen (same UI pattern as `receive.tsx:27-44`):
```
[ Public ]  [ Private ]
```
- Private toggle is **disabled** when `useWalletStore.confidentialAvailable === false` (tongoContract is `"0x0"`)
- Switching mode clears the recipient field

**Recipient input (private mode)**:
- Placeholder: `"tongo:<x>:<y>"`
- Accepts typed input (the `tongo:<x>:<y>` format shared via receive.tsx QR)
- **"Scan" button** next to the input → opens full-screen camera QR scanner modal (see below)
- Validated with `parseTongoQr()` — `null` result blocks review and shows an alert

> **Note on base58 vs tongo:<x>:<y>**: The PRD mentions "base58 or QR scan". The `tongo.address` property returns a base58 string, but `tongo.recipientId` returns `{x, y}`. The `receive.tsx` QR encodes in `tongo:<x>:<y>` format (the recipientId). The primary sharing path for recipients is QR from receive.tsx — typed input accepts that same `tongo:<x>:<y>` format. Base58 input support can be added in Cat 7 (UI polish) once Tongo docs clarify the base58 → recipientId conversion.

**"Amount will be hidden on-chain" indicator** — purple info banner shown in private mode below the amount input:
```
"Amount will be hidden on-chain via ZK proof"
```

**SDK call**: `wallet.tx().confidentialTransfer(tongo, { amount, to: parsed, sender: wallet.address }).preflight()` then `.send()`

**Stage machine** (extended from public flow):
```
"input" → "reviewing" → "generating_proof" → "sending" → "done"
```

**Preflight**: Called in `"reviewing"` stage before user confirms. Shows `"Preflight passed"` + private transfer summary. Stays in reviewing until user taps "Confirm & Send".

**QR scanner**: `expo-camera` `CameraView` in a `Modal`. `onBarcodeScanned` calls `parseTongoQr` — closes on valid match, shows `Alert.alert("Invalid QR")` on failure. Camera permission requested on first open.

**Transaction recording**: `recordTx({ type: "send", isPrivate: true, ... })` — same `TxRecord` shape, `isPrivate` flag is what drives the "Private" badge in `TransactionItem`.

**Done screen** (private): Purple `"Private — amount hidden on-chain"` badge above amount/recipient.

## 4.3 Fund & Withdraw Flows

> PRD: Fund moves STRK/ETH/USDC → confidential balance (`.confidentialFund`). Withdraw moves confidential balance → public address (`.confidentialWithdraw`).

### Fund — `app/(app)/fund.tsx` (NEW)

Stage machine: `"input" → "reviewing" → "funding" → "done"`

- Header: "Fund Private Balance" + `← Back` button (`router.back()`)
- Info text: "Move tokens from your public wallet into your private balance"
- **Token selector**: `AmountInput` component (ETH / STRK / USDC) — user picks which asset to make private
- Review: `wallet.tx().confidentialFund(tongo, { amount, sender }).preflight()` — shows confirmation card
- Confirm: calls `fundConfidential(onboard, tongo, amountStr, token)` from `lib/tongo.ts`
- Records tx: `{ type: "fund", isPrivate: true, counterparty: "self" }`
- Done: shows amount + token funded, "Back to Home" → `router.replace("/(app)/home")`

Not a tab bar item — accessed via `router.push("/(app)/fund")` from `ConfidentialBalanceCard`.

### Withdraw — `app/(app)/withdraw.tsx` (NEW)

Stage machine: `"input" → "reviewing" → "withdrawing" → "done"`

- Header: "Withdraw to Public" + `← Back` button
- Shows current `confidential.balance` at top (so user knows available amount)
- **Token selector**: `AmountInput` (ETH / STRK / USDC)
- Withdrawal destination: `wallet.address` (self), shown for transparency
- Review: `wallet.tx().confidentialWithdraw(tongo, { amount, to: wallet.address, sender }).preflight()`
- Confirm: calls `withdrawConfidential(onboard, tongo, amountStr, token, wallet.address)` from `lib/tongo.ts`
- Records tx: `{ type: "withdraw", isPrivate: true, counterparty: "self" }`
- Done: "Back to Home"

**Ragequit** — emergency full withdrawal, shown at bottom in "input" stage only:
- Red `bg-red-900/40 border border-red-900` button labelled "Emergency Withdraw All"
- Sub-label: "Drains entire private balance to your public wallet"
- Destructive `Alert.alert` with Cancel + "Withdraw All" buttons before executing
- Calls `ragequit(onboard, tongo, wallet.address)` from `lib/tongo.ts`
- Records tx with `amount: "ALL"`, `token: "CONF"`

### Tab bar — `app/(app)/_layout.tsx` (EXTEND)

```tsx
<Tabs.Screen name="fund" options={{ href: null }} />
<Tabs.Screen name="withdraw" options={{ href: null }} />
```
Prevents fund/withdraw from appearing as tab bar items.

## 4.5 ZK Proof UX

> PRD: "Generating proof..." loading state, "Verifying on-chain...", error states for insufficient balance and invalid recipient address.

### Loading states in `app/(app)/send.tsx`

**`"generating_proof"` stage** (new, shown between reviewing and sending):
```tsx
<ActivityIndicator color="#7B5EA7" />
<Text className="text-purple-400 mt-2 font-medium">Generating ZK proof...</Text>
<Text className="text-neutral-500 text-xs mt-1">This takes less than a second</Text>
```
> The Starkzap SDK's `.send()` handles proof generation + submission atomically. The `"generating_proof"` stage is set immediately before calling `sendPrivate()`, then switches to `"sending"` once `.send()` returns (proof done, tx submitted).

**`"sending"` stage** (private mode copy):
```tsx
<Text className="text-neutral-400 mt-2">Verifying on-chain...</Text>
```

### Error states

| Error | How triggered | Display |
|---|---|---|
| Insufficient balance | Preflight fails: `result.ok === false` | `Alert.alert("Transaction would fail", result.reason)` → back to `"input"` |
| Invalid recipient address | `parseTongoQr()` returns `null` | `Alert.alert("Invalid address", "Enter a valid Tongo address (tongo:<x>:<y>)")` |
| Invalid QR scan | `parseTongoQr(data)` returns `null` | `Alert.alert("Invalid QR", "This is not a valid Tongo address QR code.")` |
| Tx rejected / thrown | Any `catch` in send/fund/withdraw | `Alert.alert("Transaction failed", e.message)` → back to `"input"` |

### `hooks/useConfidentialBalance.ts` (EXTEND)

Upgraded to support the balance display that Cat 4 depends on:

```typescript
// 15s polling (matching useBalance.ts pattern)
useEffect(() => {
  fetchState();
  const id = setInterval(fetchState, 15_000);
  return () => clearInterval(id);
}, [fetchState]);

// Rollover with loading state and error handling
const [rollingOver, setRollingOver] = useState(false);
// setRollingOver(true) → execute rollover → finally setRollingOver(false)
// catch(e) → Alert.alert("Rollover failed", ...)

return { refetch: fetchState, needsRollover, rollover, rollingOver };
```

## SDK Types Reference

```typescript
// TxBuilder confidential methods
wallet.tx().confidentialFund(tongo, { amount, sender }): TxBuilder
wallet.tx().confidentialTransfer(tongo, { amount, to, sender }): TxBuilder
wallet.tx().confidentialWithdraw(tongo, { amount, to, sender }): TxBuilder
// All three support .preflight() and .send()

// ragequit and rollover are NOT on TxBuilder — use direct:
tongo.ragequit({ to, sender }): Promise<Call[]>  → wallet.execute(calls)
tongo.rollover({ sender }): Promise<Call[]>       → wallet.execute(calls)

// State
tongo.getState(): Promise<{ balance: bigint, pending: bigint, nonce: bigint }>
tongo.recipientId: { x: BigNumberish, y: BigNumberish }  // serialize as "tongo:<x>:<y>"

// Amount
Amount.parse(amountStr: string, token: Token): Amount   // user input → Amount
amount.toBase(): bigint                                  // for display of confidential units
```

## Edge Cases & Guards

| Scenario | Guard |
|---|---|
| `tongoContract === "0x0"` | `ConfidentialBalanceCard` early-returns "unavailable". Private toggle in send.tsx disabled. |
| Typed invalid Tongo address | `parseTongoQr` → `null` → Alert, blocked from review |
| Scanned invalid QR | Same — Alert in `onBarcodeScanned` |
| Camera permission denied | `requestCameraPermission()` checked before opening scanner |
| Preflight failure | `result.ok === false` → Alert with reason, reset to `"input"` |
| Zero / empty amount | `Amount.parse` throws → caught → Alert |
| Double-submit | Stage machine: action buttons only render in the correct stage |
| Rollover in progress | `disabled={rollingOver}` on rollover button |
| Ragequit accident | Destructive Alert confirmation required before executing |
